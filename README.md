<h1> Лабораторная работа №2.1 "Источники задач и контракты" </h1>
<hr>
<p>Программа содержит утилиты для взятия задач из разных источников</p>

<p>Поддерживает следующие команды:</p>
<ul>
    <li> types - отображает типы источников задач </li>
    <li> get-tasks - вывести задачи из указанного источника. Для ввода имени файла нужен флаг -f/--file </li>
</ul>

<h2> Описание модулей </h2>
<ul>
     <li>main.py - входная точка программы. </li>
     <li>cli.py - пользовательский интерфейс.  </li>
     <li>task.py - класс задачи. </li>
     <li>task_source.py - класс источника задач, заданный как протокол. </li>
     <li>source/ - модули источников и регистра. </li>
     <li>utilities/ - модули утилит: исключения, ID по умолчанию, логгер </li>
</ul>

<h2> Алгоритмы </h2>

<h2> Класс задачи Task </h2>
    <p>Простой пользовательский класс с двумя строковыми полями: id(ID задачи) и payload(собственно задача)</p>

<h2> Класс источника задач TaskSource </h2>
    <p>Протокол источника задач. Содержит метод get_tasks, являющийся определяющим для всех источников</p>

<h2> Класс APISource </h2>
    <p>Источник задач, имитирующий работу API через заглушку. Обладает методами stub, который возвращает список задач в виде словаря, и get_tasks, который преобразует этот массив в итератор задач(уже в виде экземпляров класса), используя ленивые вычисления</p>

<h2> Класс GeneratorTask </h2>
    <p>Источник задач, являющийся генератором этих самых задач. Обладает методом get_tasks, который генерирует для каждой задачи ID и выбирает payload задачи из списка(находится в модуле src/constants.py) и возвращает итератор задач(уже в виде экземпляров класса), используя ленивые вычисления</p>

<h2> Класс JSONLinesSource </h2>
    <p>Источник задач, читающий их из файла JSON Lines. Обладает методами parser, который парсит JSON-строки, и get_tasks, который читает файл и создаёт итератор задач, используя ленивые вычисления</p>

<h2> Фабрика источников </h2>
    <p>Состоит из слоаря-регистра, задекорированных функций, создающих экземпляр ресурса, и декоратора этих функций, который помещает эти функции в регистр. Затем в модуле src/cli.py во время импорта все источники добавляются в регистр, что даёт возможность добавления новых источников без изменения существующего кода(нужно только импортировать источник в cli.py)</p>

<h2> Система логирования </h2>
    <p>В файле src/config.py хранится словарь конфигурации логера. Далее в модуле src/utilities/logger.py создаётся экземпляр логгера, который импортируется в другие модули по необходимости. Логи сохраняются в файл debug.log, который периодически самоочищается при достижнении размера 3*5 МБ(удаляются файлы со старыми логами). Если файл debug.log отсутствует, создаётся новый, иначе логи записываются в существующий debug.log.</p>
